1. INTRODUÇÃO

Em dias contemporâneos as evoluções tecnológicas oportunizam acesso a informação, novas formas de comunicação, escolarização e inclusão a pessoas com deficiência, proporcionando acessibilidade e mais autonomia no seu cotidiano. A tecnologia assistiva surge como uma estratégia que visa possibilitar melhorias na qualidade de vida e aprendizagem em especial a pessoas com deficiência visual (Bruno; Nascimento,2019). Pensando nisto, foi idealizado um projeto que auxiliasse pessoas com deficiência visual em seu dia a dia, através de um dispositivo que ajuda no deslocamento, utilizando sensores para detectar obstáculos. O trabalho surgiu da seguinte problemática: que meios os deficientes visuais dispõem para enfrentar dificuldades de locomoção e risco de acidentes em seu cotidiano? Diante desta questão, com o objetivo de impactar positivamente sobre essa problemática, propõe-se a disseminação do projeto: Sistema de navegação assistiva para deficientes visuais, visando mais independência, segurança e inclusão para estas pessoas.

2. OBJETIVO:

Desenvolver um dispositivo de baixo custo que ajude pessoas com deficiência visual a se deslocarem por ambientes internos, utilizando sensores para detectar obstáculos e fornecer feedback auditivo em tempo real.

3.1 MATERIAIS NECESSÁRIOS PARA A CONSTRUÇÃO DO KIT:

• Kit Arduino Uno; • Notebook; • Sensor ultrassônico (HC-SR04) para detecção de obstáculos; • Buzzer ou speaker para feedback sonoro; • Fios de conexão; • Protoboard; • Alimentação (bateria ou fonte de 9V). • Fita métrica (para ajustar o alcance dos sensores); • Caixa ou suporte para acomodar os componentes;

3.2 FUNCIONAMENTO DO PROJETO E APRESENTAÇÃO DE CADA ITEM.

Detecção de obstáculos: O sensor ultrassônico é colocado na frente do usuário, e ele emite ondas sonoras que são refletidas por objetos à sua frente. O sensor calcula a distância até o obstáculo com base no tempo que a onda leva para voltar. 1. Feedback sonoro: O Arduino processa os dados do sensor e emite diferentes sons através do buzzer ou speaker para alertar o usuário sobre a proximidade dos obstáculos: • Som contínuo: Quando o obstáculo está muito próximo (por exemplo, a menos de 20 cm). • Som intermitente (pulsante): Para distâncias intermediárias (por exemplo, entre 20 cm e 1 metro). • Silêncio ou som leve: Quando o caminho está livre ou a distância é grande (mais de 1 metro). 2. Funcionamento: O usuário pode carregar o dispositivo (por exemplo, em uma mochila ou cinto) e se movimentar pela sala ou corredor. À medida que se aproxima de um obstáculo, o feedback sonoro muda, ajudando a pessoa a ajustar sua direção.

4. RESULTADOS

Esse projeto é simples, prático e pode ser uma excelente forma de demonstrar como a tecnologia pode ser usada para melhorar a qualidade de vida de pessoas com deficiência visual. Por meio das ações pretendidas com o trabalho, o projeto proposto visa unir a teoria estudada durante os dias trabalhados com a montagem do dispositivo, relacionando assim a teoria de conceitos e do conteúdo teórico, a prática (na montagem do kits e reconhecimento de cada componente que será utilizado). Além disso, possibilita também uma visão diferenciada observando a relevância do projeto e interdisciplinaridade entre as demais disciplinas, reconhecendo também a questão principal que é a inclusão e o respeito a pessoas com necessidades especiais. A montagem do kit também possibilitou um novo recurso para deficientes visuais, no qual se destacou como positiva, podendo também haver novas melhorias Com os estudos e pesquisas realizados nestes meses podemos aprimorar o kit para Versões futuras, sendo elas: Sistema de Navegação Assistiva para Deficientes Visuais, podemos adicionar duas funcionalidades importantes: localização do deficiente e um botão de SOS para situações de emergência.

MELHORIAS NO PROJETO PARA O FUTURO

1. Localização do Deficiente Visual Para adicionar a funcionalidade de localização do deficiente, podemos usar um módulo GPS. Esse módulo permitirá rastrear a posição geográfica do usuário e, por exemplo, enviar a localização via Bluetooth para um dispositivo móvel ou para um servidor. Componentes necessários: • Módulo GPS (como o NEO-6M): Para rastrear a localização do usuário. • Módulo Bluetooth (HC-05 ou HC-06): Para enviar a localização para um smartphone ou outro dispositivo. • Bateria adicional: Para alimentar os novos componentes. Como funciona: • O módulo GPS irá ler as coordenadas de latitude e longitude, e, a cada intervalo de tempo, o Arduino pode atualizar essas coordenadas. • Com o módulo Bluetooth, o Arduino pode enviar essas coordenadas a um aplicativo no smartphone, permitindo que uma pessoa responsável saiba a localização do deficiente em tempo real. 2. Botão SOS O botão SOS serve como uma medida de segurança adicional para alertar quando o usuário estiver em perigo ou precisar de ajuda. Esse botão pode enviar um sinal de emergência para uma pessoa responsável, um centro de apoio ou até enviar uma mensagem automática. Componentes necessários: • Botão de emergência: Um botão simples (push-button) para enviar um sinal de SOS. • Módulo de comunicação (Bluetooth ou GSM): Para enviar o alerta. • Alimentação adicional: Para suportar o módulo de comunicação extra. Como funciona: • Quando o botão SOS for pressionado, o Arduino envia um sinal de alerta via Bluetooth ou GSM. • Caso use o Bluetooth, o sinal pode ser enviado para um aplicativo no smartphone, alertando alguém próximo sobre a situação de emergência. • Se usar o módulo GSM, é possível enviar um SMS ou realizar uma chamada telefônica automaticamente. 3. Integração das Melhorias • Localização com GPS: Você pode integrar a localização com a funcionalidade do botão SOS. Por exemplo, ao pressionar o botão SOS, o sistema pode enviar não apenas o alerta, mas também a localização do usuário em tempo real para um responsável. • Feedback Audível para o SOS: Quando o botão SOS for pressionado, você pode adicionar um feedback auditivo (como um som contínuo no buzzer) para confirmar que o alerta foi enviado. Considerações Finais: • Alimentação: Como estamos adicionando novos módulos (GPS, Bluetooth, e possivelmente GSM), o consumo de energia será maior. Certifique-se de usar uma fonte de alimentação adequada, como baterias recarregáveis. • Testes: Realize testes em diferentes cenários para garantir que o sistema funcione corretamente, tanto no aspecto de localização quanto no envio do alerta SOS. Benefícios das Melhorias: • Segurança: O botão SOS aumenta a segurança do usuário, permitindo uma forma rápida de pedir ajuda. • Autonomia: A localização em tempo real proporciona mais independência para o deficiente visual, permitindo que seus responsáveis saibam onde ele está. • Tecnologia acessível: A utilização de sensores, GPS e Bluetooth pode ser uma solução simples e eficaz para melhorar a qualidade de vida de pessoas com deficiência visual. Com essas melhorias, o projeto se torna ainda mais completo e pode ser muito útil na vida cotidiana de deficientes visuais, proporcionando maior segurança e autonomia.
